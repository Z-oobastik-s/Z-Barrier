# Zbarrier

Плагин для **Paper**: настройка границ миров (`WorldBorder`) по каждому миру отдельно, блокировка выхода за пределы, предупреждения у края, опциональное сжатие и рост границы по таймеру.

**Версия Minecraft:** `1.21` (сборка под API **Paper 1.21.4**).  
**Java:** `21`.  
Сервер: **Paper** (рекомендуется; используется Adventure / MiniMessage из стека Paper).

---

## Установка

1. Соберите JAR (`gradle shadowJar`) или возьмите готовый артефакт.
2. Положите `Zbarrier.jar` в папку `plugins`.
3. Запустите сервер один раз, отредактируйте `plugins/Zbarrier/config.yml` и при необходимости `messages.yml`.
4. Выполните `/zbarrier reload` или перезапуск сервера.

Файлы плагина: `config.yml`, `messages.yml` (префикс и тексты чата в формате MiniMessage).

---

## Возможности

- Отдельная граница на мир: размер, центр (режим **AUTO** от спавна или **MANUAL** по координатам).
- Ограничения: откат движения, запрет телепорта за границу, откат транспорта (настраивается по миру).
- Сообщение игроку при приближении к краю (дистанция и кулдаун в конфиге).
- **Dynamic:** периодическое увеличение или уменьшение диаметра в заданных пределах (с белым списком миров в глобальных настройках).
- Шаблон **`world-defaults`** и короткие секции `worlds.<имя>: {}` для переиспользования настроек.
- Автодобавление секций для новых загруженных миров (можно отключить).
- Режим отладки в лог и команда **`debugwhere`** для проверки мира и границы с точки зрения сервера.

Подробные пояснения к ключам конфига смотрите в комментариях внутри **`config.yml`**.

## Команды

Короткий алиас: **`/zb`** (то же, что `/zbarrier`).

| Команда | Назначение |
|--------|------------|
| `/zbarrier` или `help` | Подсказка по командам из `messages.yml` |
| `reload` | Перечитать `config.yml` и `messages.yml`, применить границы, перезапустить dynamic |
| `debugwhere` | Только игрок: имя мира, ключ измерения, размер и центр границы этого мира |
| `setsize <мир> <диаметр>` | Записать размер и применить к миру |
| `setcenter <мир> <x> <z>` | Центр вручную + применение |
| `setcenterhere [мир]` | Только игрок: центр по вашим X/Z (мир по умолчанию - текущий) |
| `autocenter <мир>` | Режим AUTO (центр от спавна мира) |
| `enable` / `disable <мир>` | Включить или выключить обработку мира плагином |
| `warning <мир> <блоки>` | Дистанция предупреждения до края |
| `dynamic <мир> grow\|shrink <шаг> <сек> <min> <max>` | Включить динамику и запустить задачу |
| `dynamicstop <мир>` | Выключить динамику и остановить задачу |

При **`settings.plugin-enabled: false`** доступны только **`reload`** и **`debugwhere`**, остальные подкоманды отвечают сообщением об отключённом плагине.

## Права

| Право | По умолчанию | Заметка |
|-------|----------------|---------|
| **`zbarrier.admin`** | OP | Проверяется на все команды; без него плагин отвечает об отсутствии прав |
| `zbarrier.reload` | OP | Объявлено в `plugin.yml`; отдельной проверки в коде нет, на практике используйте **`zbarrier.admin`** |

## Настройки (кратко)

Все ключи и сценарии расписаны в **`config.yml`**. Краткий ориентир:

| Блок | Смысл |
|------|--------|
| **`debug`** | Подробный лог применения границ и диагностика |
| **`settings.plugin-enabled`** | Главный выключатель логики плагина |
| **`settings.enforce-barrier`** | Слушатели движения / транспорта и откат |
| **`settings.prevent-outside-teleport`** | Отмена телепорта за границу |
| **`settings.pullback-distance`** | Насколько внутрь оттягивать при выходе за лимит |
| **`settings.min-size` / `max-size`** | Жёсткие пределы диаметра |
| **`settings.dynamic-allowed-worlds`** | Список миров или `*` для допуска dynamic |
| **`world-defaults`** | Общий шаблон для миров |
| **`worlds.<имя>`** | Переопределения под конкретный мир |

Тексты игрокам правятся в **`messages.yml`** (MiniMessage, плейсхолдеры вида `<world>`).

## Сборка

Требуется JDK **21** и Gradle. Из корня проекта:

```bat
gradle shadowJar
```

Готовый JAR: имя артефакта **`Zbarrier.jar`** (см. `build.gradle.kts`).
