# Zbarrier

Плагин для **Paper**: отдельная граница мира (`WorldBorder`) на каждый мир, откат движения и транспорта у края, запрет телепорта наружу, предупреждение игроку, опционально автоматическое сжатие или рост границы по таймеру.

| **Какое ядро нужно?** | **Paper 1.21.11.** Плагин собирается против `paper-api` **1.21.11-R0.1-SNAPSHOT** (см. `build.gradle.kts`). |
| **Какая Java?** | **21.** |
| **Подойдёт ли Paper 1.21.4, 1.21.8 и т.п.?** | **Не заявлено.** Другие патчи 1.21.x могут заработать, но автор не обещает: API и поведение отличаются. Ставьте **1.21.11**, если нужна гарантия по задумке проекта. |
| **Подойдёт ли Spigot без Paper?** | **Нет** в смысле официальной поддержки: в коде используется стек Paper / Adventure (сообщения в `messages.yml`). |
| **Сервер 1.20 или ниже?** | **Нет.** В `plugin.yml` указано `api-version: '1.21'`, ниже 1.21 ядро плагин не загрузит нормально. |
| **Bedrock, прокси, одиночка?** | Плагин только для **Paper-мира** на Java-сервере. |

Итого одной строкой: **Minecraft 1.21.11 + Paper 1.21.11 + Java 21.**

## Что умеет плагин

- Свой **размер и центр границы** для каждого мира из конфига; центр от **спавна** (`AUTO`) или **координат** (`MANUAL`).
- **Удержание игрока** внутри зоны: откат шага, откат транспорта, отмена телепорта за границу (включается по миру и глобальным флагам).
- **Предупреждение** в чат при приближении к краю (дистанция и кулдаун в конфиге).
- **Dynamic:** по расписанию увеличивать или уменьшать диаметр в заданных пределах; список разрешённых миров в конфиге.
- Шаблон **`world-defaults`** и короткие записи **`worlds.<имя>: {}`**, автодобавление секций для новых миров (опционально).
- **`/zbarrier debugwhere`** и режим **`debug`** в конфиге для проверки, что сервер видит за мир и границу.

Полный разбор всех ключей и сценариев лежит в **комментариях внутри `config.yml`** в репозитории.

## Установка

1. Скачайте или соберите **`Zbarrier.jar`**.
2. Положите в **`plugins`** на сервере **Paper 1.21.11**.
3. Запустите сервер, отредактируйте **`plugins/Zbarrier/config.yml`** и при необходимости **`messages.yml`**.
4. В консоли или в игре с правами: **`/zbarrier reload`** (или перезапуск сервера).

## Команды

Алиас: **`/zb`** = **`/zbarrier`**. Нужно право **`zbarrier.admin`** (по умолчанию OP).

Если в конфиге **`settings.plugin-enabled: false`**, работают только **`reload`** и **`debugwhere`**; остальные подкоманды ответят, что плагин выключен.

| Синтаксис | Кратко |
|-----------|--------|
| `/zbarrier` / `help` | Список подсказок из `messages.yml` |
| `reload` | Перечитать конфиги, применить границы, перезапустить dynamic |
| `debugwhere` | Только игрок: мир, ключ измерения, размер и центр границы |
| `setsize <мир> <диаметр>` | Записать размер, применить к миру |
| `setcenter <мир> <x> <z>` | Ручной центр + применение |
| `setcenterhere [мир]` | Только игрок: центр в вашей точке X/Z |
| `autocenter <мир>` | Центр от спавна мира (`AUTO`) |
| `enable <мир>` / `disable <мир>` | Включить или выключить мир в логике плагина |
| `warning <мир> <блоки>` | Дистанция предупреждения до края |
| `dynamic <мир> grow\|shrink <шаг> <сек> <min> <max>` | Включить динамику и таймер |
| `dynamicstop <мир>` | Выключить динамику и таймер |

| Право | По умолчанию | Примечание |
|-------|----------------|------------|
| **`zbarrier.admin`** | OP | Реально проверяется на все команды. |
